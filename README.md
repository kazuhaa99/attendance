# Attendance Dashboard

Дашборд учета посещений сотрудников. Отображает визиты через пропускную систему ElPass, отсутствия из BDAS/Impala, рабочие часы и аналитику.

---

## Быстрый старт

### Требования

- **Docker** + **Docker Compose** (рекомендуется)
- Или: **Node.js 20+**, **Python 3.12+**

### 1. Настройка окружения

```bash
cp .env.example .env
```

Отредактируйте `.env`:

| Переменная | Описание | Пример |
|---|---|---|
| `ELPASS_BASE_URL` | URL API пропускной системы | `http://pass.telecom.kz/api` |
| `ELPASS_EMAIL` | Логин для ElPass | `dashboard@elpass.kz` |
| `ELPASS_PASSWORD` | Пароль для ElPass | `D@shboard1!` |
| `PAGE_SIZE` | Размер страницы при загрузке визитов | `100` |
| `MAX_ROWS` | Максимум строк визитов | `50000` |
| `MAX_CONCURRENT` | Параллельных запросов к ElPass | `10` |
| `CORS_ORIGINS` | Разрешенные домены (JSON-массив) | `["http://localhost:5173"]` |
| `IMPALA_ENABLED` | Включить загрузку отсутствий из Impala | `true` / `false` |
| `IMPALA_HOST` | Хост Impala/BDAS | `bdas-worker-02.bdpak.telecom.kz` |
| `IMPALA_PORT` | Порт Impala | `21050` |
| `IMPALA_USER` | Пользователь Impala | |
| `IMPALA_PASSWORD` | Пароль Impala | |
| `IMPALA_TABLE` | Таблица отсутствий | `drb.drb_assanali_ndkt_cal` |

### 2. Запуск через Docker (рекомендуется)

```bash
docker compose up --build
```

- Frontend: http://localhost:5173
- Backend: http://localhost:8000
- API docs: http://localhost:8000/api/docs

### 3. Запуск без Docker

**Backend:**
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

### 4. Продакшен

```bash
cp .env.example .env.prod
# Отредактируйте .env.prod с продовыми значениями
docker compose -f docker-compose.prod.yml up --build -d
```

В проде frontend собирается в статику и раздается через Nginx на порту 3001.

## Интерфейс

### Шапка (Header)

- **Выбор дат** — диапазон `от` / `до` для загрузки визитов
- **Кнопка обновления** — перезагрузить данные с сервера
- **Время обновления** — когда данные были загружены последний раз

### Панель фильтров (FilterBar)

| Фильтр | Описание |
|---|---|
| **Поиск по сотруднику** | Текстовый поиск по имени. Применяется ко всем вкладкам: визиты, отсутствия, часы. Поиск с задержкой 350мс для оптимизации |
| **Без имени** | Показать только записи без привязанного имени карты |
| **Зоны** | Фильтр по зоне прохода (например, Kaisar Plaza) |
| **Терминалы** | Фильтр по конкретному устройству |
| **Подразделения** | Фильтр по подразделению из группы карты |
| **Направление** | Все / Вход / Выход |
| **Сбросить** | Сбросить все активные фильтры (показывает счетчик активных) |

### KPI-карточки (KPIRow)

Верхняя строка с ключевыми метриками:

- **Всего визитов** — общее число записей за период
- **Уникальных карт** — количество разных посетителей
- **Входов / Выходов** — по направлению `isOut`
- **Устройств** — сколько терминалов зафиксировало проходы
- **Сотрудников** — общее количество (пришедшие + отсутствующие)

При активном фильтре по имени дополнительно показываются:
- **Часов отработано** — суммарные часы выбранного сотрудника
- **Первый вход / Последний выход** — за период

### Графики

- **Timeline Chart** — количество визитов по дням. Клик по столбцу фильтрует на конкретный день
- **Direction Chart** — соотношение входов и выходов (круговая диаграмма). Клик фильтрует по направлению

### Статистика рабочего времени (WorktimeStats)

4 карточки-кнопки с подсчетом аномалий:

| Карточка | Что показывает |
|---|---|
| **Ранний приход** | Вход до 08:00 |
| **Поздний приход** | Вход после 09:30 |
| **Ранний уход** | Выход до 17:30 |
| **Поздний уход** | Выход после 19:00 |

Клик по карточке фильтрует таблицу визитов, показывая только соответствующие записи.

### Часы работы (HoursPanel)

Таблица рабочих часов сотрудников:

- **Строки** — сотрудники, отсортированные по суммарным часам (от большего к меньшему)
- **Колонки** — дни из выбранного периода
- **Разбивка по неделям** — при диапазоне больше 1 недели дни группируются по неделям (Пн-Вс) с итогом за каждую неделю в колонке «Нед.»
- **Цветовая индикация**:
  - Зеленый — 8+ часов
  - Оранжевый — 6-8 часов
  - Красный — менее 6 часов
- **Расчет**: последний выход минус первый вход минус 1 час (обед)
- **Клик по имени** — переход к визитам сотрудника в таблице
- **Пагинация** — по 25 человек на страницу

### Отсутствия (AbsencePanel)

Данные из Impala/BDAS (кадровая система):

- **Типы**: Отпуск, Больничный, Командировка и др.
- **Фильтр по типу** отсутствия
- **Фильтр аномалий** — сотрудники, которые числятся отсутствующими, но зафиксированы на проходной
- **Итоговая строка** — количество записей по каждому типу
- **Клик по строке** — фильтрует визиты по этому сотруднику

> Если `IMPALA_ENABLED=false` — панель не отображается.

### Таблица визитов (VisitsTable)

Детальная таблица всех проходов:

| Колонка | Описание |
|---|---|
| Время | Дата и время прохода |
| Сотрудник | Имя с карты. Клик — фильтр по имени |
| Карта | Номер карты |
| Направление | Вход (зеленый) / Выход (красный) |
| Терминал | Устройство. Клик — фильтр по терминалу |
| Зона | Зона прохода. Клик — фильтр по зоне |
| Подразделение | Из группы карты. Клик — фильтр по подразделению |
| Статус | Метка если сотрудник числится отсутствующим |

## API эндпоинты

| Метод | URL | Описание |
|---|---|---|
| `GET` | `/api/visits` | Визиты за период. Параметры: `date_from`, `date_to`, `name`, `zone`, `terminal`, `direction`, `no_name` |
| `GET` | `/api/groups` | Справочник подразделений (код → название) |
| `GET` | `/api/absences` | Отсутствия за период. Параметры: `date_from`, `date_to` |
| `POST` | `/api/absences/refresh` | Принудительно обновить кэш отсутствий |
| `GET` | `/api/absences/cache-status` | Статус кэша отсутствий |
| `GET` | `/api/health` | Проверка здоровья сервиса |
| `GET` | `/api/docs` | Swagger UI (автогенерация FastAPI) |

### Кэширование отсутствий

Данные из Impala кэшируются до следующих 22:00 (время обновления витрины). Для принудительного обновления: `POST /api/absences/refresh`.

## Решение проблем

| Проблема | Решение |
|---|---|
| «Сервер пропускной системы недоступен» | Проверьте `ELPASS_BASE_URL` в `.env`. Убедитесь что URL доступен из контейнера/сервера. Протокол `http` vs `https` важен |
| Визитов 0, но предупреждения нет | Проверьте диапазон дат — возможно данных за этот период нет |
| Отсутствия не загружаются | Установите `IMPALA_ENABLED=true` и заполните `IMPALA_*` переменные. Проверьте доступность Impala-хоста |
| Docker Desktop требует WSL | Запустите `wsl --install` от администратора, затем перезагрузите ПК |
| Порт 5173/8000 занят | Остановите другие процессы или измените порты в `docker-compose.yml` |
| CORS ошибка в браузере | Добавьте URL фронтенда в `CORS_ORIGINS` в `.env` |
